## Operations
There are three dominant operations within the Understandings System: *Create*, *Split* and *Merge*. Knowing when to use each of these - especially *Create* - is key to running an Understandings System.

### Split
A split is where what was known as one Understanding is now multiple. The split operation will also create the aggregate taxon to represent all the known components of a split.

### Merge
Merging is for when what were thought to be multiple taxa are only one. A merge with a single input

### Create
This operation creates an entirely novel Understanding. A novel Understanding is one which has never existed in any sense within the area prior to this point. The vast majority of these cases will be taxa moving into the region from outside.
