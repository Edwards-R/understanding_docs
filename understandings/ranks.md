# Ranks in the Understandings System
The Understanding System does not prescribe a set of ranks to use, as doing so would be needlessly constrictive. The ruling with regards to ranks within the Understandings System is that every entity must have an entry at every given rank. No ranks, of those present in the system, may be skipped or omitted. If 'subspecies' is a rank, every single Understanding must reach the rank of subspecies.

All ranks must be in a strict hierarchy, where every rank (other than the Capstone rank) belong to a rank above it.

## Capstone rank
In order to provide a cap to the hierarchy of ranks, the Capstone rank exists. This rank links to itself and sits at the very top of the rank structure. It is recommended to use a rank name of 'Capstone' for this rank, to avoid any confusion.

## Subsetting ranks
It is permitted to 'subset' ranks within the Understandings System. Rather than having to start at the highest rank (currently generally accepted to be Domain at the time of writing), it is possible to start at a lower rank e.g. Superfamily.

## Example
|Rank    |Parent Rank|
|--------|-----------|
|Capstone|Capstone   |
|Family  |Capstone   |
|Genus   |Family     |
|Species |Genus      |
