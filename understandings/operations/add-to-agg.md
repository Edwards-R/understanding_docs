**WIP**


    Oh god why.
    - Have an aggregate

    - It's not from a clean split of any existing current Understanding
    - Just part of the amorphous blob
    - How to place these into the mix

## Options
- ~~Merge on aggregates?~~
- Merge on Agg1 components to make Agg2 and redirect Agg1 to Agg2?
- ~~Insert new component into Agg1 invisibly~~

## Thoughts
- What it contains has changed, therefore it needs to be new
- Don't want to unlock the 'no aggregates' rule on the split/merge
- This is REALLY edge case
- Answer is to make a new aggregate and redirect the old to the ne then?
    - Don't like it, but I don't think I'll ever like this
    - This is SUPER edge case


## Rules
- Can only be used to add a current Understanding to an existing aggregate
- Must be total confusion with **more than one** other component of the aggregate
    - If confusion with only one component, then it's a split of that component

## Method
- Select an existing agg
    - All components of the agg must be current Understandings
